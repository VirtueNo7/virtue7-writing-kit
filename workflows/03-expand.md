# Workflow 3 - Expand

## Input

Identify:

- target artifact;
- current resolution;
- target resolution;
- approved canonical source;
- audience and format;
- evidence policy;
- constraints.

If the canonical source is missing, create or recover it before expanding.

## Expansion brief

Before drafting, state:

```yaml
source_artifact: ""
source_resolution: ""
target_artifact: ""
target_resolution: ""
functions_added: []
canonical_dependencies: []
evidence_required: []
may_transform: []
must_preserve: []
approval_state: "Draft"
```

## Draft

Add only named functions: clarify, explain, demonstrate, compare, apply, practise, or adapt.

When expanding to a chapter, a strong default movement is:

`Orient -> Teach -> Demonstrate -> Decode -> Compare -> Apply -> Resolve -> Compress`

The sequence may change when the material demands it, but the responsibilities may not simply vanish beneath an atmospheric opening.

## Validate

- compare against kernel and unit record;
- verify sources and quotations;
- identify newly introduced claims;
- compress the output;
- run the round-trip test;
- present findings before approval.
