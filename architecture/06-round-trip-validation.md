# 6. Semantic Round-Trip Validation

Each project defines an invariant set `I`. A derivative output is valid when a careful reviewer can compress it back into a statement equivalent to the approved kernel with respect to `I`.

`Compress(Output) ≡ Kernel, with respect to invariant set I`

Equivalence does not require identical wording. It requires preserved meaning.

## Round-trip questions

1. Does the derivative preserve the same direction or central question?
2. Does it preserve the causal or logical mechanism?
3. Are definitions still bounded and distinguishable?
4. Are prohibited interpretations still prohibited?
5. Is the promised reader value still present?
6. Has a decisive qualification, counterexample, uncertainty, or safety boundary been omitted?
7. Can the output be compressed to the approved paragraph, sentence, decision line, and kernel formula?
8. If not, is the derivative wrong, or has the canonical source legitimately evolved?

## Drift classes

- **Definition drift:** a key term changes meaning.
- **Direction drift:** the output points toward a different conclusion.
- **Causal drift:** the mechanism is replaced by correlation, assertion, or atmosphere.
- **Evidence drift:** interpretation is presented as fact.
- **Boundary drift:** a non-goal or qualification disappears.
- **Narrative drift:** the story becomes the thesis.
- **Brand drift:** kit or demonstration language leaks into the user's work.

Every failed round trip produces a visible finding and a proposed correction. The AI may propose; the human decides whether the canonical source or derivative changes.
